Twitter's Custom API REST version 1.0

http://localhost:8080/

Introduccion

Este servicio usa las librerias de Twitter4j para consumir la API publica de Twitter y con esta informacion se crea una API REST personalizada.

Twitter's Custom API REST

/tweets get

get

Este servicio permite listar los tweets provenientes de la libreria de streaming de Twitter4j

get /tweets

Este servicio permite listar los tweets provenientes de la libreria de streaming de Twitter4j

  • Response

HTTP status code 200

Body

Media type: application/json

Type: array of Tweet

Items: Tweet

  • id: required(integer)
  • screenName: required(string)
  • text: required(string)
  • place: required(string)
  • valid: required(boolean)

Example:

[
    {
      "id": 0,
      "screenName": "User123",
      "text": "My recent tweet",
      "place": "Bogota, Colombia",
      "valid": false
    }
]

/tweets/valid/{id} put

put

Este servicio permite actualizar el estado de validacion de un tweet, para ello debe ingresar en la URI el codigo del tweet.

put /tweets/valid/{id}

Este servicio permite actualizar el estado de validacion de un tweet, para ello debe ingresar en la URI el codigo del tweet.

  • Request
  • Response

URI Parameters

  • id: required(string)

HTTP status code 201

Body

Media type: application/json

Type: object

Properties
  • id: required(integer)
  • screenName: required(string)

  • place: required(string)

/tweets/trends get

get

Este servicio permite listar las tendencias mas relevantes alrededor del mundo.

get /tweets/trends

Este servicio permite listar las tendencias mas relevantes alrededor del mundo.

  • Response

HTTP status code 200

Body

Media type: application/json

Type: array of Trend

Items: Trend

  • name: required(string)
  • url: required(string)
  • query: required(string)
  • tweetVolume: required(integer)

Example:

[
  {
    "name": "#RHOA",
    "url": "http://twitter.com/search?q=%23RHOA",
    "query": "%23RHOA",
    "tweetVolume": 16476
  }
]
